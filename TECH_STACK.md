## 4. Supabase Integration Rules

### 4.1 Client Instantiation

**Rule:** Never call `createClient()` directly. Always use the auto-imported composable from `@nuxtjs/supabase`.

```typescript
// ✅ CORRECT — use in composables, pages, and stores
const supabase = useSupabaseClient()

// ❌ WRONG — never do this
import { createClient } from '@supabase/supabase-js'
const supabase = createClient(url, key)
```

### 4.2 Authentication Pattern

```typescript
// composables/useAuth.ts
export const useAuthComposable = () => {
  const supabase = useSupabaseClient()
  const user = useSupabaseUser()  // Reactive ref from @nuxtjs/supabase

  const getRole = (): 'admin' | 'cashier' => {
    return (user.value?.user_metadata?.role as 'admin' | 'cashier') ?? 'cashier'
  }

  const login = async (email: string, password: string) => {
    const { data, error } = await supabase.auth.signInWithPassword({ email, password })
    if (error) throw error
    return data
  }

  const logout = async () => {
    await supabase.auth.signOut()
    navigateTo('/login')
  }

  return { user, getRole, login, logout }
}
```

### 4.3 Database Query Patterns

**Rule:** All queries must be typed using the generated database types. Regenerate types after every schema change with:
```bash
npx supabase gen types typescript --project-id [PROJECT_ID] > types/database.ts
```

```typescript
// ✅ CORRECT — typed query
const supabase = useSupabaseClient<Database>()

const { data, error } = await supabase
  .from('products')                           // Table name as string literal
  .select('id, nama_produk, harga_jual, stok_sekarang, is_active, umkm_id')
  .eq('session_date', todayDate)
  .eq('is_active', true)
  .gt('stok_sekarang', 0)
  .order('nama_produk', { ascending: true })

if (error) throw error
// data is fully typed as products row[]
```

**Rule:** Always destructure `{ data, error }` and handle `error` before using `data`.

```typescript
// ✅ CORRECT — always check error
const { data, error } = await supabase.from('umkm').select('*')
if (error) {
  console.error('[fetchUmkm]', error.message)
  throw new Error(error.message)
}
return data

// ❌ WRONG — ignoring error
const { data } = await supabase.from('umkm').select('*')
return data  // data could be null on error
```

### 4.4 RPC Call Pattern

```typescript
// Calling Supabase database functions
const { data, error } = await supabase.rpc('complete_transaction', {
  p_session_id:       sessionId,
  p_cashier_id:       userId,
  p_nominal_diterima: nominalDiterima,
  p_cart_items:       cartItems.map(item => ({
    product_id: item.product_id,
    qty:        item.qty,
    harga_jual: item.harga_jual,
    // NOTE: harga_asli is NOT sent from client — RPC reads it from DB
  }))
})

if (error) {
  // Parse the Postgres error message for user-friendly display
  throw new Error(error.message)
}
// data is the JSONB return value from the function
```

### 4.5 Realtime Subscription Pattern

```typescript
// Always store the channel reference for cleanup
let productsChannel: RealtimeChannel | null = null

const subscribeToProducts = (sessionDate: string) => {
  productsChannel = supabase
    .channel(`products-${sessionDate}`)        // Unique channel name per session
    .on(
      'postgres_changes',
      {
        event: 'UPDATE',
        schema: 'public',
        table: 'products',
        filter: `session_date=eq.${sessionDate}`,
      },
      (payload) => {
        const updated = payload.new as ProductRow
        productStore.updateProductStock(updated.id, updated.stok_sekarang)
        cartStore.updateStockWarning(updated.id, updated.stok_sekarang > 0)
      }
    )
    .subscribe()
}

// Always unsubscribe on component unmount
onUnmounted(() => {
  if (productsChannel) {
    supabase.removeChannel(productsChannel)
    productsChannel = null
  }
})
```

### 4.6 Data Fetching Rules

| Context | Method | Rule |
|---|---|---|
| Page initial data | `useAsyncData` in `setup()` | For SSR-compatible async data (even in SPA mode) |
| Store actions | `async` Pinia actions | Trigger from `onMounted` in page |
| Real-time updates | Supabase Realtime channel | Subscribed in `onMounted`, unsubscribed `onUnmounted` |
| Mutations | Direct Supabase calls in store actions | Never mutate from component directly |

```typescript
// pages/pos.vue — correct data loading pattern
const productStore = useProductStore()

// Use useAsyncData for page-level data
const { pending, error } = await useAsyncData(
  'today-products',
  () => productStore.fetchTodayProducts(),
  { server: false }  // SPA mode — client only
)

onMounted(() => {
  productStore.subscribeRealtime()
})

onUnmounted(() => {
  productStore.unsubscribeRealtime()
})
```

## 5. Component Architecture

### 5.1 Component Rules

- **All components use `<script setup lang="ts">`** — no Options API, no `defineComponent`
- **Props must be typed** with `defineProps<{...}>()`
- **Emits must be typed** with `defineEmits<{...}>()`
- **No business logic in components** — components call store actions; they do not call Supabase directly
- **No inline styles** — Tailwind classes only

### 5.2 Component Template

```vue
<!-- components/pos/ProductCard.vue -->
<script setup lang="ts">
import type { ProductCashierView } from '~/types/app'

// Props — always typed
const props = defineProps<{
  product: ProductCashierView
  isInCart: boolean
  cartQty: number
}>()

// Emits — always typed
const emit = defineEmits<{
  addToCart: [product: ProductCashierView]
}>()

// Store access — composables only
const cartStore = useCartStore()

// Computed
const isDisabled = computed(() =>
  props.product.stok_sekarang <= 0 ||
  props.cartQty >= props.product.stok_sekarang
)

const formattedPrice = computed(() =>
  useCurrencyFormat(props.product.harga_jual)
)

// Handlers
const handleTap = () => {
  if (!isDisabled.value) {
    emit('addToCart', props.product)
  }
}
</script>

<template>
  <button
    :disabled="isDisabled"
    class="..."
    @click="handleTap"
  >
    <!-- template content -->
  </button>
</template>
```

## 6. Pinia Store Patterns

### 6.1 Store Template (Composition API Style)

```typescript
// stores/products.ts
import { defineStore } from 'pinia'
import type { RealtimeChannel } from '@supabase/supabase-js'
import type { ProductCashierView } from '~/types/app'

export const useProductStore = defineStore('products', () => {
  // --- State ---
  const products = ref<ProductCashierView[]>([])
  const isLoading = ref(false)
  const error = ref<string | null>(null)
  let realtimeChannel: RealtimeChannel | null = null

  // --- Getters ---
  const activeProducts = computed(() =>
    products.value.filter(p => p.is_active && p.stok_sekarang > 0)
  )

  const getByUmkm = (umkmId: string) =>
    products.value.filter(p => p.umkm_id === umkmId)

  // --- Actions ---
  const fetchTodayProducts = async () => {
    const supabase = useSupabaseClient()
    const { todayDate } = useSessionDate()

    isLoading.value = true
    error.value = null

    try {
      const { data, error: fetchError } = await supabase
        .from('products_cashier_view')    // Uses the RLS-safe view
        .select('*')
        .eq('session_date', todayDate.value)
        .order('nama_produk')

      if (fetchError) throw fetchError
      products.value = data ?? []
    } catch (e: unknown) {
      error.value = e instanceof Error ? e.message : 'Unknown error'
      throw e
    } finally {
      isLoading.value = false
    }
  }

  const updateProductStock = (productId: string, newStock: number) => {
    const product = products.value.find(p => p.id === productId)
    if (product) product.stok_sekarang = newStock
  }

  const subscribeRealtime = () => {
    const supabase = useSupabaseClient()
    const { todayDate } = useSessionDate()

    realtimeChannel = supabase
      .channel(`products-stock-${todayDate.value}`)
      .on('postgres_changes', {
        event: 'UPDATE',
        schema: 'public',
        table: 'products',
        filter: `session_date=eq.${todayDate.value}`,
      }, (payload) => {
        updateProductStock(payload.new.id, payload.new.stok_sekarang)
      })
      .subscribe()
  }

  const unsubscribeRealtime = () => {
    if (realtimeChannel) {
      const supabase = useSupabaseClient()
      supabase.removeChannel(realtimeChannel)
      realtimeChannel = null
    }
  }

  return {
    // State
    products, isLoading, error,
    // Getters
    activeProducts, getByUmkm,
    // Actions
    fetchTodayProducts, updateProductStock,
    subscribeRealtime, unsubscribeRealtime,
  }
})
```

### 6.2 Store Rules

- Use **composition API style** (`defineStore('id', () => {...})`) — not options style
- Store ID must match the file name (e.g., `useProductStore` in `stores/products.ts` has id `'products'`)
- Never call `useSupabaseClient()` at store module level — only inside action functions
- All async actions must set `isLoading` and handle errors via `error` ref
- Stores do not navigate — emit events or let components handle navigation after store action resolves

## 8. PWA & Service Worker Rules

### 8.1 Caching Strategy

| Asset Type | Strategy | Cache Name | Notes |
|---|---|---|---|
| App shell (HTML/JS/CSS) | `CacheFirst` | `app-shell` | Versioned; update on deploy |
| Supabase REST API | `NetworkFirst` | `supabase-api` | 24h fallback cache |
| Supabase Auth | `NetworkOnly` | — | Never cache auth endpoints |
| Supabase Realtime (WSS) | Not cached | — | WebSocket — cannot be cached |
| Static assets (icons, fonts) | `CacheFirst` | `static-assets` | Long TTL |

### 8.2 Offline Queue (IndexedDB)

```typescript
// composables/useOfflineQueue.ts
import { openDB } from 'idb'

const DB_NAME = 'omk-pos-offline'
const STORE_NAME = 'pending-transactions'

interface PendingTransaction {
  id:                string          // Local UUID
  timestamp:         string          // ISO string
  session_id:        string
  cashier_id:        string
  nominal_diterima:  number
  cart_items:        CartItem[]
  status:            'pending' | 'synced' | 'failed'
  error_message?:    string
}

export const useOfflineQueue = () => {
  const getDb = () => openDB(DB_NAME, 1, {
    upgrade(db) {
      db.createObjectStore(STORE_NAME, { keyPath: 'id' })
    }
  })

  const enqueue = async (transaction: Omit<PendingTransaction, 'status'>) => {
    const db = await getDb()
    await db.put(STORE_NAME, { ...transaction, status: 'pending' })
  }

  const getPending = async (): Promise<PendingTransaction[]> => {
    const db = await getDb()
    const all = await db.getAll(STORE_NAME)
    return all.filter(t => t.status === 'pending').sort(
      (a, b) => new Date(a.timestamp).getTime() - new Date(b.timestamp).getTime()
    )
  }

  const markSynced = async (id: string) => {
    const db = await getDb()
    const record = await db.get(STORE_NAME, id)
    if (record) await db.put(STORE_NAME, { ...record, status: 'synced' })
  }

  const markFailed = async (id: string, errorMessage: string) => {
    const db = await getDb()
    const record = await db.get(STORE_NAME, id)
    if (record) await db.put(STORE_NAME, { ...record, status: 'failed', error_message: errorMessage })
  }

  return { enqueue, getPending, markSynced, markFailed }
}
```

### 8.3 Sync on Reconnect

```typescript
// composables/useNetworkStatus.ts
export const useNetworkStatus = () => {
  const isOnline = ref(typeof navigator !== 'undefined' ? navigator.onLine : true)
  const { getPending, markSynced, markFailed } = useOfflineQueue()
  const cartStore = useCartStore()

  if (typeof window !== 'undefined') {
    window.addEventListener('online', async () => {
      isOnline.value = true
      await syncPendingTransactions()
    })
    window.addEventListener('offline', () => {
      isOnline.value = false
    })
  }

  const syncPendingTransactions = async () => {
    const pending = await getPending()
    if (pending.length === 0) return

    const { addToast } = useToast()
    let syncedCount = 0

    for (const tx of pending) {
      try {
        await cartStore.submitTransaction({
          sessionId:       tx.session_id,
          cashierId:       tx.cashier_id,
          nominalDiterima: tx.nominal_diterima,
          cartItems:       tx.cart_items,
        })
        await markSynced(tx.id)
        syncedCount++
      } catch (e: unknown) {
        const msg = e instanceof Error ? e.message : 'Unknown error'
        await markFailed(tx.id, msg)
      }
    }

    addToast({
      type: syncedCount === pending.length ? 'success' : 'warning',
      message: `${syncedCount} dari ${pending.length} transaksi offline berhasil disinkronkan.`
    })
  }

  return { isOnline }
}
```

## 13. Forbidden Patterns

The following patterns will be rejected in code review. The AI Agent must never generate code that contains these:

### 13.1 Never Access `harga_asli` from Cashier Context

```typescript
// ❌ FORBIDDEN — cashier must never see harga_asli
const { data } = await supabase
  .from('products')
  .select('*, harga_asli')    // Never select harga_asli for cashier role

// ✅ CORRECT — query the safe view
const { data } = await supabase
  .from('products_cashier_view')
  .select('*')
```

### 13.2 Never Mutate Stock Directly from Client

```typescript
// ❌ FORBIDDEN — direct stock mutation from frontend
await supabase
  .from('products')
  .update({ stok_sekarang: newStock })
  .eq('id', productId)

// ✅ CORRECT — always via complete_transaction RPC
await supabase.rpc('complete_transaction', payload)
```

### 13.3 Never Create Transactions Outside the RPC

```typescript
// ❌ FORBIDDEN — direct insert bypasses atomic stock deduction
await supabase.from('transactions').insert({ ... })
await supabase.from('transaction_details').insert({ ... })
// (These are two non-atomic operations — race condition risk)

// ✅ CORRECT — single atomic RPC call
await supabase.rpc('complete_transaction', payload)
```

### 13.4 Never Use localStorage for Cart or Session Data

```typescript
// ❌ FORBIDDEN
localStorage.setItem('cart', JSON.stringify(cartItems))

// ✅ CORRECT — use Pinia store (in-memory) for cart state
// ✅ CORRECT — use IndexedDB (via idb) for offline transaction queue
```

### 13.5 Never Hard-Code Session Date

```typescript
// ❌ FORBIDDEN
.eq('session_date', '2025-06-15')
.eq('session_date', new Date().toISOString().split('T')[0])  // UTC — wrong timezone!

// ✅ CORRECT — use the Jakarta-aware utility
import { getTodayJakarta } from '~/utils/date'
.eq('session_date', getTodayJakarta())
```

### 13.6 Never Bypass Middleware for Admin Routes

```typescript
// ❌ FORBIDDEN — role check inside the page component
// pages/admin/dashboard.vue
const user = useSupabaseUser()
if (user.value?.user_metadata?.role !== 'admin') navigateTo('/pos')

// ✅ CORRECT — declare middleware in definePageMeta
definePageMeta({
  middleware: ['auth', 'admin']   // Runs before the page renders
})
```

### 13.7 Never Compute Financial Totals in the Frontend

```typescript
// ❌ FORBIDDEN — frontend summing transaction_details
const totalRemittance = transactions.flatMap(t => t.details)
  .reduce((sum, d) => sum + d.subtotal_harga_asli, 0)

// ✅ CORRECT — use the get_session_financial_summary RPC
const { data } = await supabase.rpc('get_session_financial_summary', {
  p_session_id: sessionId
})
// data.total_remittance is computed by DB aggregation
```
